# 四半期切替の手順

クォーターをまたぐタイミングは、plan-app の中で **TF 構成が変わる** 唯一の節目です。
ここを丁寧にやっておくと、翌Qが始まってすぐ会議ノートを書き始められます。

## 全体の流れ

```
前Q末週                   切替週                     翌Q初週
 ────────── ────────────────────── ──────────►
 ①〜③通常運用    クォーター計画           次QへTF移動・確定
                  （AI×全データ分析）       メンバー周知
                                          翌Q①会議ノート開始
```

## ステップ1：前Qの最終週（前Q末週）

- 通常通り ①会議ノート → ②セッション → ③レポート を回す
- ③レポート確定で、Q末の総括が翌Q初①の引き継ぎメモに流れる

> Q末週は「振り返り深め」のチェックインに切り替えると、クォーター計画の素材が厚くなります。

## ステップ2：クォーター計画（切替週）

OKRモード → **計画**タブ → **クォーター計画**：

1. **対象 KR**・**計画対象クォーター**・**注力したい課題・テーマ**（任意）を入力
2. **「計画を始める」** をクリック
3. AI が前Qの全データ（会議ノート・セッション・宣言・分析）を読み込み、**最初に総括分析**を返す
4. その後は **対話形式**で TF 案を詰めていく
5. 仕上がったら TF 一覧を確定（**継続／変更／廃止／新設**）

## ステップ3：管理画面で次Qの構造を反映（管理者）

設定 → **タスクフォース**（上部のクォータータブを次Qに切り替えて作業）：

1. **TF の追加・更新・廃止**
   - 新設：次Qのタブのまま新しい TF を作成（作成時のクォーターが自動的にセットされる）
   - 変更：既存 TF の name/leader を更新
   - 廃止：is_deleted（履歴は残る）
2. **既存TFを次Qへ移動**
   - 前Qから継続するTFはTF行の「移動」操作で次Qへ切り替える（TF自体は削除されない）
   - 1つのTFが同時に複数クォーターに属することはない（移動すると前Qからは外れる）

> ❗ **次Qへの移動・新設を忘れると**、翌Q初週の会議ノートでそのTFが対象クォーターに出てこず（または前Qのタブに残ったままになり）混乱します。**必ず翌Q月曜の前日までに**完了させてください。

## ステップ4：メンバーに変更を周知（管理者）

クォーター切替は **全員が認識を揃える必要がある**節目です。最低限：

- 新設 TF とリーダー
- 廃止 TF
- 注力テーマの変更
- 役割の変更（KR代表など）

を文書で共有してください。テンプレ：

```
件名：[plan-app] 〇Q運用開始のお知らせ
今期の Objective： <変更があれば>
KR1： <タイトル>
  TF1： <name> / リーダー <name>
  TF2： <name> / リーダー <name>
  ...
KR2： ...
変更点：
  - 廃止：TF <name>（理由：…）
  - 新設：TF <name>（理由：…）
今期の注力課題： …
```

## ステップ5：翌Q初週

通常運用に戻ります。注意点：

- ①会議ノートの「TFの説明・その期のテーマ」欄に **その Q の方針** を改めて書き直す
- 前Q末の③レポートからの引き継ぎメモが大きくなりがちなので、**今週の論点だけを抜き出す**意識で

## トラブル対応

### 翌Q月曜になったのに TF の移動が間に合わない
暫定対応：
1. その週は前Qの TF をそのまま使う（会議ノートのクォーターセレクタで前Qに切替）
2. 並行して管理者が設定 → タスクフォースで翌QへのTF移動・新設を完了
3. 翌週から翌Q TF で運用開始

### TF を廃止したのに会議ノートに出てくる
- 該当 TF の `is_deleted` が true になっているか確認
- 設定 → タスクフォースで、そのTFのクォーターが表示中のQのままになっていないか確認

### 前Qのレポートが見当たらない
履歴オーバーレイ（右上 🕐 履歴）から週切替で遡れます。
