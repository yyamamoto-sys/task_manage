# 用語集

plan-app・OKR運用で使う用語の**唯一の定義場所**です。本文で説明を増やさず、ここを引いてください。

## OKR の基本

### Objective
組織として「いつまでに、なぜ、何を成したいか」を定性的に宣言する1〜2文。期間は通常1年（plan-app では `period` で持つ）。

### KR（Key Result）
Objective の達成度を測る**結果指標**。「達成したかどうか」が外形的に判断できる粒度で書く。plan-app では Objective に複数 KR がぶら下がる構造。

### TF（タスクフォース）
KR を実現するために結成される**実働ユニット**。1つのKRに複数TFがぶら下がる。TF はクォーターごとに編成・組替えされる（`quarterly_kr_task_forces` で割り当て管理）。

### ToDo
TF が「やる」と決めた中間アウトプット。タスクの集合の単位。

### タスク
ToDo を分解した実行可能な作業単位。担当者・期日・状態（done/doing/todo）を持つ。

## 週次サイクル

### ① 会議ノート
KR代表がKR会議で記入する、TFごとの「仮説／事実／次の一手／進捗％」記録。**KR×週で1件**、その中にTF分のセクションが並ぶ。

### ② セッション記録 & 分析
KR1・KR2合同で行うチェックイン／ウィンセッションの議事録をアップロードし、AI が KR ごとに振り分け・詳細分析・宣言抽出を一気に行うステップ。

### ③ レポート作成
②の分析結果を素材に、KRごとの週次レポートを作成・確定するステップ。確定したレポートは翌週の①に「引き継ぎメモ」として自動で反映される。

### 引き継ぎメモ
①会議ノート上部に表示される、前週確定レポートと最新③分析の要点を自動生成したメモ。今週の会議の前提情報として使う。

## セッション種別

### チェックイン
週初に「今週の宣言」を立てる場。signal（🟢🟡🔴）と簡潔なコメント、誰が何をいつまでにやるかを記録。

### ウィンセッション
週末に先週の宣言の結果を振り返る場。result（達成／一部達成／未達）と学び・次の一手を記録。

### freeform
チェックイン・ウィン以外の自由形式OKR議論。

## ロール

### KR代表
そのKRの責任者。週次の会議ノートを書き、KR の状態を最も把握している人。

### ファシリテーター
チェックイン・ウィンセッションの進行役。議事メモを取り、セッション記録のアップロードまで行うことが多い。

### 管理者
Objective・KR・TF・メンバー登録、クォーターのTF割り当てなどの設定を行う人。

### メンバー
TF に所属し、ToDo・タスクを動かす人。

## 技術用語（簡易）

### Supabase
plan-app のバックエンド（PostgreSQL ＋ Edge Functions）。OKR の各種データを保管。

### Edge Function `ai-consult`
Anthropic Claude API を呼び出すためのサーバ側関数。各種AI機能（分析・抽出・レポート生成）はすべてこれを経由。

### intent
AI呼び出し時にどの用途で使うかを表すタグ。利用量集計とプロンプト切替に使う（例: `kr-session-extract`, `kr-quarter-plan`）。
