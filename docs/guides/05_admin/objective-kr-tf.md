# Objective・KR・TF を登録する

plan-app の OKR 構造は **3階層**です：

```
Objective（1件・通年）
  ├─ KR（複数・通年）
  │   ├─ TF（クォーター毎に編成）
  │   │   ├─ ToDo
  │   │   │   └─ タスク
```

期初に Objective→KR→TF の順で登録し、メンバーが使い始められる状態を作ります。

## 全体の流れ

1. Objective（1件）
2. KR（Objective に複数ぶら下げる）
3. メンバー
4. TF（KR にぶら下げる・初期Q分）
5. 各TFのクォーターを確認（TF一覧のクォータータブ）
6. パイロットメンバーに通知

## 1. Objective を登録

**設定 → Objective**：

- **title**：「いつまでに、なぜ、何を成したいか」を1〜2文で
- **purpose**：背景・なぜそれをやるか
- **period**：通常は1年分（例「2026」）

> Objective は同時に1件まで。期が変わる時は新規登録ではなく更新で OK（履歴が残る）。

## 2. KR を登録

**設定 → KR**：

- **title**：達成したかどうかが外形的に判断できる結果指標
- 数値 KR は **数値・単位・期日**を含めると AI 分析の精度が上がる
- 1 Objective に対し 2〜5件程度

## 3. メンバーを登録

**設定 → メンバー管理**：

- **short_name**：2〜4字（アバターや表示で使う）
- **full_name**：氏名
- **email**：ログイン用
- **role**：admin / facilitator / kr-rep / member などのタグ（運用上の目印・権限制御には現状未使用）

## 4. TF を登録

**設定 → タスクフォース**：

- **name**：TF名（業務内容がわかる短いラベル）
- **kr_id**：どの KR にぶら下げるか
- **tf_number**：1〜N（表示順）
- **leader_member_id**：TFリーダー（任意）

1 KR に 2〜5 TF が目安。多すぎると会議ノートが重くなります。

## 5. TFのクォーターを確認（重要）

**設定 → タスクフォース**の上部にある**クォータータブ（1Q〜4Q）**：

各TFは1つのクォーターに属します（TF自身の「クォーター」欄）。タブを切り替えると、そのQに属するTFだけが表示されます。

- 既存TFを別 Q に**移動**できる（TF行の操作）。TF自体を削除せずに継続・変更が可能
- クォーター未設定（レガシー）のTFは自動的に「現在のクォーター」として扱われる
- **クォーターの切替前に必ず**、次Qで使うTFがそのタブに揃っているか確認する（移動 or 新規作成）

> 1Qを設定した後、2Qに切り替わるタイミングで「次Qは何をやるか」をクォーター計画機能で議論してから、TFをそのQへ移動・新設する。

## 6. 動作確認

- メンバー1人として実際にログインし、OKRモードで自分の KR を選んで TF が表示されることを確認
- ダッシュボードで KR と紐づくタスクが見えること
- ① 会議ノートが空で開けること

## トラブル

### TFが重複して見える
表示中のクォーターと異なる Q の TF が紛れている状態。設定 → タスクフォースでそのTFのクォーターを確認・移動すれば解消します。

### KRにTFをぶら下げたのに会議ノートに出てこない
- 対象クォーターと違う Q に設定されている可能性（設定 → タスクフォースのクォータータブで確認）
- 該当 TF が `is_deleted = true` になっている可能性

### メンバーがログインできない
メンバー登録の email がログイン時の選択画面に出ているか確認。出ていなければ登録失敗の可能性。
