# 新しい部署・メンバーを追加する

plan-app は部署（グループ）ごとにデータが分離されています。他部署の**プロジェクト・タスク・メンバー情報は互いに見えません**。この分離を保ったまま新しい部署・メンバーを増やすための、唯一の正しい手順です。

## 大原則：URLを送るだけでは使えない

新しく使い始める人に plan-app の URL を送っても、それだけでは何も見えません。**先に全社スーパー管理者が、その人のメールアドレスを使ってメンバー登録を済ませておく**必要があります。

理由：本人がサインアップ（新規登録）した直後は、まだどの部署にも所属していません。plan-app はメールアドレスの一致でメンバーとその所属部署を紐づける設計のため、**先に登録がないと本人が何をしても紐づきません**。この場合、サインアップ後に表示される「アクセス権がありません」という画面の案内に従って、管理者へメンバー登録を依頼してもらってください。

## 手順1：新しい部署を追加する（全社スーパー管理者のみ）

1. 管理画面（⚙ 設定）→ **グループ**セクションを開く
2. 「＋ 部署を追加」ボタンをクリック（全社スーパー管理者にのみ表示されます）
3. 部署名を入力する。あわせて、その部署の最初のメンバー（表示名・略称・メールアドレス）を入力すると、部署作成と同時にその人が**部署管理者**として登録されます
4. 保存する

## 手順2：メンバーを登録する（全社スーパー管理者、または当該部署の部署管理者）

1. 管理画面 → **メンバー**セクションを開く
2. 「＋ メンバーを追加」から、表示名・略称・**メールアドレス（必須）**・所属部署を入力する
3. 必要に応じて「部署管理者」「全社スーパー管理者」の権限も設定する
4. 保存する

> **メールアドレスの一致が紐づけの鍵です。** ここで登録した `email` と、本人がSupabase Authでサインアップする際に使うメールアドレスが完全に一致していないと、本人がログインしても自動的にはマッチングされません。

## 手順3：本人にサインアップしてもらう

1. 手順1・2で登録したメールアドレスで、plan-app の URL からサインアップ（新規登録）してもらう
2. 確認メールのリンクを開いてログインすると、登録済みのメールアドレスと自動的に照合され、正しい部署のメンバーとしてアプリが使えるようになります

## 補足：真の初回セットアップ（システムに誰もいない場合）

上記は「既に部署・メンバーが1件以上存在する」通常運用の手順です。plan-app を**史上初めて**立ち上げる場合（membersが1件も存在しない状態）に限り、最初にサインアップした人にセットアップウィザードが表示され、部署名と自分自身の情報を入力するだけで、その部署と自分自身（部署管理者・全社スーパー管理者）が自動的に作成されます。これは一度きりの特別な経路で、通常の部署追加・メンバー追加には使いません。

## 注意点

- メンバー登録・部署作成ともに、ログインしただけの未登録ユーザーには操作できません（管理者権限が必要です）
- 部署を間違えて登録した場合は、メンバー編集画面から所属部署を付け替えられます（全社スーパー管理者、または異動元・異動先どちらかの部署管理者）
- アクティブなメンバーが1人でもいる部署は、全社スーパー管理者以外は削除できません（誤操作防止）
