# 部署ごとのTeams通知チャンネル設定

毎週月曜、期限が近い・超過しているタスクをTeamsに自動投稿する「週次期限通知」は、部署（グループ）ごとに別々のチャンネルへ届けることができます。

## 設定の流れ

1. 通知を届けたい部署専用のTeamsチャンネルを用意する（無ければ先に作成する）
2. Power Automateで、そのチャンネル用の投稿先URLを発行する（下の「Power Automate側の手順」を参照）
3. このアプリの管理画面 → グループ → 対象の部署を編集 →「Teams Webhook URL」欄に、発行されたURLを貼り付けて保存する

これだけで、次回の週次通知からその部署専用のチャンネルに届くようになります。URLを設定していない部署は、これまで通り全社共通のチャンネルにまとめて届きます。

## Power Automate側の手順（ゼロから作らず複製する）

新しくフローを組み立てる必要はありません。既にある「タスク管理アプリ｜Webhookアラート_EGG」というフローを複製し、投稿先チャンネルだけ変更するのが簡単です。

1. Power Automate（make.powerautomate.com）を開き、「マイフロー」→「自分と共有」から「タスク管理アプリ｜Webhookアラート_EGG」を開く
2. 画面上部の「名前をつけて保存」をクリックする
3. フロー名を部署がわかる名前に変更する（例：タスク管理アプリ｜Webhookアラート_〇〇部）→ 保存する
4. 作成されたコピーを開き、「チャットまたはチャネルでメッセージを投稿する」というアクションを探す
5. その中の「チーム」「チャネル」を、通知を届けたい部署のチーム・チャンネルに変更して保存する
6. コピー直後はフローがオフになっていることがあるので、オンになっているか確認する（オフなら「オンにする」を押す）
7. フロー詳細画面に戻り、トリガー部分（Webhook要求を受信したときに）を開くとURLが表示されるのでコピーする

このURLが、アプリ側で設定する「Teams Webhook URL」です。

## 注意点

- Webhook URLの設定・変更は全社スーパー管理者のみ操作できます
- 複製したフローの中身（メンションの解決やタスクの振り分けロジック）は変更しないでください。変更するのは投稿先チャンネルの指定だけで十分です
- 心配な場合は、開発担当に「テストモードで確認してほしい」と伝えてください。実際にTeamsへ投稿せずに、生成される内容だけを確認できる安全な確認方法があります
