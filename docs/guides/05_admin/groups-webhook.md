# 部署ごとのTeams通知チャンネル設定

毎週月曜、期限が近い・超過しているタスクをTeamsに自動投稿する「週次期限通知」は、部署（グループ）ごとに別々のチャンネルへ届けることができます。

## 設定の流れ

1. このアプリの管理画面 → グループ → 対象の部署を編集 →「Power Automate用テンプレートをダウンロード」ボタンでテンプレートファイル（.zip）を取得する
2. Power Automateで、そのテンプレートから部署専用の投稿先URLを発行する（下の「Power Automate側の手順」を参照）
3. 発行されたURLを、同じ編集画面の「Teams Webhook URL」欄に貼り付けて保存する

これだけで、次回の週次通知からその部署専用のチャンネルに届くようになります。URLを設定していない部署は、これまで通り全社共通のチャンネルにまとめて届きます。

## Power Automate側の手順（テンプレートを取り込む）

1. Power Automate（make.powerautomate.com）を開き、左メニュー「マイフロー」→ 上部の「インポート」→「パッケージのインポート (レガシ)」を選ぶ
2. ①でダウンロードしたzipファイルをアップロードする
3. インポート内容の確認画面が出る。「Microsoft Teams 接続」の項目で、自分自身のTeams接続を選択する（「新規作成」でも、既存の接続がある場合はそれを選んでもよい）
4. 「インポート」を実行する
5. インポートされたフローを開き、「チャットまたはチャネルでメッセージを投稿する」というアクションを探す
6. その中の「チーム」「チャネル」を、通知を届けたい部署のチーム・チャンネルに変更して保存する
7. フローがオフになっている場合はオンにする
8. フロー詳細画面に戻り、トリガー部分（Webhook要求を受信したときに）を開くとURLが表示されるのでコピーする

このURLが、アプリ側で設定する「Teams Webhook URL」です。

## 注意点

- Webhook URLの設定・変更、テンプレートのダウンロードはログイン済みユーザーのみ利用できます
- テンプレートの中身（メンションの解決やタスクの振り分けロジック）は変更しないでください。変更するのは投稿先チャンネルの指定だけで十分です
- 心配な場合は、開発担当に「テストモードで確認してほしい」と伝えてください。実際にTeamsへ投稿せずに、生成される内容だけを確認できる安全な確認方法があります
