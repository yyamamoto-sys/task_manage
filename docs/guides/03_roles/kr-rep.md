# KR代表ガイド

KR代表は、その KR の **状態を最も把握している人**です。週次サイクルでは①と③を主に担当し、②には KR を代表して参加します。

## あなたの週次タスク

| いつ | 何を | 所要 |
|---|---|---|
| 週初 | **①会議ノート**を記入（KR会議で／会議前に） | 15分〜 |
| 週中 | **②セッション**に参加（合同／グループメンバー全体）| 1時間 |
| 週末 | **③レポート**を確定（AI初稿→手修正→確定） | 10分〜 |

## ①会議ノートの書き方

KR会議の主目的は**事実を整える**ことです。AI は事実から仮説と次の一手を出してくれるので、書き手の負担は最小限。

- **事実を入れる場所**：「② 実際に起きたこと（事実・反応）」
  - **評価や解釈を混ぜないこと**。「うまくいった」ではなく「Aさんから○○という反応があった」
- **仮説は別欄**：「① 先週動かした前提・仮説」
- **数字**は「④ 現在のプロセス状態（％）」に。理由も併記

引き継ぎメモは前週の③レポート確定時に自動生成されます。届いていなければ「↻ 引き継ぎメモを自動生成」を試してください。

→ 詳細： [① 会議ノート](../02_modes/okr/01_meeting-note.md)

## ②セッションでのKR代表の動き

合同セッションでは、各 KR の代表が **その KR の状況をひと言で**説明する役割です。
分析と宣言抽出は AI がやってくれるので、その後の確認・修正もあなたがリードします。

- AI が抽出した宣言が**自分のKRの分しか拾えていない**と感じたら、画面上で追加可
- 「その他の論点」が紛れている場合も補強できます

→ 詳細： [② セッション記録 & 分析](../02_modes/okr/02_session.md)

## ③レポートの確定

- AI 初稿はあくまで叩き台。**1〜2か所は必ず手直しが入る**前提でOK
- 「来週への引き継ぎポイント」は特に丁寧に。これが翌週①の引き継ぎメモになります
- **確定（finalized）するまで**翌週に流れません。忘れずに

→ 詳細： [③ レポート作成](../02_modes/okr/03_report.md)

## 困った時の優先順位

1. 画面右上の `?` ボタン（その画面の使い方）
2. このガイド
3. [FAQ](../06_troubleshooting/faq.md)
4. 山本さんに連絡

## あなたが避けたい失敗

- 引き継ぎメモを読まずに会議ノートを書く → 前週の学びが流れない
- 「事実」欄に評価を書く → AIの分析精度が落ちる
- レポートを `draft` のまま放置 → サイクルが切れる
