# ファシリテーター・ガイド

ファシリテーターは、チェックイン／ウィンセッションの**進行と記録**を担当します。
plan-app の②セッション機能を中心に、サイクルを止めない役割です。

## あなたの週次タスク

| いつ | 何を | 所要 |
|---|---|---|
| 週初 | チェックインの進行 | 30〜60分 |
| 週中／週末 | ウィンセッションの進行 | 30〜60分 |
| セッション直後 | 議事メモを plan-app の **②セッション** に投入 | 5〜10分 |

会議自体の進行はあなたの会議体のやり方で構いません。
**plan-app に取り込むのはセッション後の議事メモ**だけです。

## セッション当日の動き

### 1. 議事メモを取る
形式は問いません：
- Teams のレコーディング書き起こし（VTT）
- 自分でメモった Word/テキスト
- 配布資料の PDF（補助）

### 2. plan-app に取り込む
1. **OKRモード** → サブタブ **「② セッション記録&分析」**
2. モードを **合同（複数KR一括）** にする（既定）
3. 対象 KR を複数選択（KR1＋KR2 など）
4. 議事メモを貼り付け or ファイルアップロード
5. **「AI に分析させる」** をクリック（スクロール無しで押せる位置に固定）

### 3. レビュー
AI 結果は KR ごとにタブが切られて出てきます。各 KR タブ＋「全体（Objective）」タブで確認。

- **シグナル**（🟢🟡🔴）と**コメント**が妥当か
- **宣言**（誰が・何を・いつまでに）が漏れていないか
  - 1時間会議なら KR ごとに複数出るはず
  - 足りなければ手動で追加
- **前週宣言の結果**が正しく反映されているか（ウィンセッションの場合）

### 4. 保存
保存すると以下が DB に書かれます：
- kr_sessions（セッション本体）
- kr_declarations（KR別の宣言）
- okr_analyses（scope: kr / objective の2レベル）

## VTT ファイルの扱い

- Teams のレコーディングから出る `.vtt` をそのままアップロード可
- タイムスタンプは Claude が無視するので問題なし
- 話者がある程度わかる場合、議事メモ側で「●●：…」のように整えておくと宣言の主体抽出精度が上がります

## 単一KRモードを使う時

- 個別 KR の深堀り議論があった場合
- 過去の会議を後追いで記録する場合
- それ以外は基本「合同」推奨

## あなたが避けたい失敗

- セッション後、議事メモを plan-app に入れずに放置 → ③レポートと翌週①が空っぽに
- AI 分析を確認せず保存 → 振り分けがズレたまま履歴に残る
- 「全体（Objective）」タブを見ずに保存 → 横断的な示唆が拾えない
