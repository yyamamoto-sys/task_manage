# OKR週次サイクル全体像

plan-app の OKR モードは、**毎週のリズム**を作るための仕組みです。
3つのステップが順に回り、**前のステップの結果が次の素材になる**ように設計されています。

```
①会議ノート  ──→  ②セッション記録&分析  ──→  ③レポート作成  ──↩ 翌週の①
（KR代表）         （合同・ファシリ）            （KR代表）
```

## 全体の流れ

### ① 会議ノート（週初・KR代表）

各 KR の代表者が、KR会議で **TFごと**にその週の状況を記入。
- 仮説／事実／次の一手／進捗％／TODO を TF単位で
- **KR×週で1件**のノート。中に対象クォーターの全TFのセクションが並ぶ
- 前週ノートから「引き継ぎメモ」として下書きを自動生成可能

→ 詳細は [① 会議ノート](./01_meeting-note.md)

### ② セッション記録 & 分析（週中・合同）

KR1・KR2 合同で行うチェックイン／ウィンセッションの議事メモを貼り付け、
AI が **詳細分析 → KRごとに振り分け → 宣言抽出**を一気に実行。
- VTT・Word・PDF・テキストをそのままアップロード可
- 1時間会議なら複数の宣言を取りこぼさず拾うようプロンプト調整済
- 分析結果は KR ごと＋Objective 全体でDBに保存され、翌週以降の参照に使う

→ 詳細は [② セッション記録 & 分析](./02_session.md)

### ③ レポート作成（週末・KR代表）

②の分析結果と①のノートを素材に、KRごとに週次レポートを書く。
- 確定（finalized）すると、**翌週の①会議ノートに「引き継ぎメモ」として自動反映**
- 後から人が手修正可。承認フローはなし

→ 詳細は [③ レポート作成](./03_report.md)

### 翌週へ：循環

確定レポート＋最新③分析が、自動で次週の①会議ノートの「引き継ぎメモ」欄に流れ込みます。
これが**サイクルが切れない仕組み**です。

## サイクル進捗バー

OKRモードで KR を選ぶと、上部に **今週のどこまで進んでいるか**が表示されます：

```
[KRタイトル] ｜5/12週   ● ①会議ノート（記入済）  ›  ○ ②セッション（未）  ›  ○ ③レポート（未）
```

各ステップをクリックすればそのタブに飛びます。

## クォーターが変わったら

クォーターをまたぐと TF 構成が変わることが多いので、**クォーター計画**機能で次の Q の TF を設計してから①に入ります。
設計後は設定 → タスクフォースのクォータータブで、実際にTFを次Qへ移動・新設してください（[四半期切替の手順](../../04_workflows/quarter-rollover.md)参照）。

## なぜなぜ分析

サイクルの中で「なぜ進まないのか」が問題になった時に、**なぜなぜ**タブで掘り下げます。
KR や TF を起点に、AI が5回の「なぜ」を回します。

→ 詳細は [なぜなぜ](./05_why.md)（執筆中）
