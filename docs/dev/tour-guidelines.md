# ツアー設計ガイドライン（基本ルール）

> アプリ内オンボーディングツアー（`src/components/tour/`）の見た目・動き・文面の基準。
> 新しいステップ／ツアーを作るとき、TourProvider を改修するときは必ずこれに従う。
> 目的：ステップ間のバラつき（明度・アニメ・体裁）をなくし、初めての人に分かりやすくする。
>
> 確定日：2026-05-25（背景50% / 控えめモーション / 親しみやすく簡潔、で合意）

---

## 0. 確定した方針（トンマナの根幹）

| 項目 | 決定 |
|---|---|
| 背景の暗さ | 不透明度 **50%**（`rgba(0,0,0,0.5)`）。全ステップ共通・**一層のみ**。 |
| モーション | **控えめ・アプリと統一**。`cubic-bezier(0.16,1,0.3,1)` / 約0.2秒。 |
| 文面のトン | **親しみやすく簡潔**（ですます調・前向き・短め）。 |

---

## 1. レイヤーと背景（明度の単一化）

### 旧実装の問題
スポットライト（ターゲット強調）ステップで暗幕が二重がけだった：
4枚の DimBox（0.5）＋ ハイライト枠の `box-shadow: 0 0 0 9999px rgba(0,0,0,0.5)`。
→ 実効約0.75。中央表示ステップ（0.5一層）より暗く、ステップ間でバラついていた。

### ルール
- 暗幕は **`--tour-scrim`（= rgba(0,0,0,0.5)）一層のみ**。
- スポットライト表現は「均一の暗幕＋くり抜き＋ブランド色の枠線」で行う。
  **枠線側の追加暗転（box-shadow の黒）は使わない。**
- 中央表示ステップ（ターゲット無し）も同じ `--tour-scrim` 一層。→ どのステップでも同じ暗さ。
- 暗幕はツアー開始時に **フェードイン**（瞬間表示しない）。ステップ移動中は維持（毎回フェードし直さない）。
- 暗幕はクリックを吸収するが**ツアーを終了しない**（誤操作防止）。終了は ✕ / スキップ / Esc のみ。

### z-index（固定）
| 要素 | z-index |
|---|---|
| 暗幕（scrim / DimBox） | 9990 |
| スポットライト枠線 | 9991 |
| 吹き出し | 10000 |

---

## 2. スポットライト（くり抜き）

| 項目 | 値 |
|---|---|
| 余白（ターゲット周囲） | **8px** |
| 角丸 | `--radius-md`（10px） |
| 枠線 | `2px solid var(--color-brand)` |
| 位置・サイズの遷移 | あり（§3） |

- ターゲットは `data-tour-id="..."` で指定（CSSセレクタ・refに依存しない）。
- 表示前に `scrollIntoView` で可視化してから測位。
- パネルのスライドイン等、表示直後にサイズが変わるターゲットは表示後に数回測り直す（実装済み）。

---

## 3. モーション（控えめ・アプリと統一）

共通トークン：イージング `--tour-ease: cubic-bezier(0.16,1,0.3,1)`／時間 `--tour-duration: 0.2s`。

| 対象 | 動き |
|---|---|
| 吹き出し（出現） | フェードイン＋6px の浮き上がり、0.2s |
| 吹き出し（ステップ移動） | 位置を `top/left` で滑らかに移動（瞬間ジャンプしない）＋内容フェード |
| くり抜き枠 | `top/left/width/height` を明示遷移（`all` は使わない）、0.25s |
| 暗幕 | 開始時に opacity 0→1（0.2s）。移動中は維持 |

- **`prefers-reduced-motion: reduce` を尊重**：全アニメ・遷移を無効化（瞬間表示）。
- 既存の `.animate-*`（globals.css）と同じ言語に揃える。新規キーフレームは最小限。

## 4. 吹き出しの構造（全ステップ共通テンプレート）

```
┌─────────────────────────────┐
│ ステップ n / N            ✕ │  ← ヘッダー（進捗・終了）
│ 🎯 タイトル（絵文字1つ）      │  ← タイトル
│ 本文。3行 or 箇条書き4点まで。 │  ← 本文
│ 「何 → できること → 次に」順  │
│            [← 戻る][スキップ][次へ →]│  ← フッター
└─────────────────────────────┘
```

| パーツ | 仕様 |
|---|---|
| ヘッダー左 | `ステップ {n} / {N}`（tertiary・10px）。※旧「👋 {タイトル}」固定は廃止 |
| ヘッダー右 | ✕（`aria-label="ツアーを閉じる"`） |
| タイトル | 13〜14px・太字・**絵文字は1つだけ**（意味のあるもの）・〜20字 |
| 本文 | 12px・line-height 1.6・secondary色・**3行 or 箇条書き4点まで** |
| フッター | ← 戻る（最初は非表示）／スキップ／次へ →（最後は「完了」）。主ボタン＝ブランド色 |

| 見た目 | 値 |
|---|---|
| 最大幅 | 360px（`calc(100vw - 24px)`） |
| 余白 | 14px 16px |
| 角丸 | `--radius-lg`（16px） |
| 影 | `--shadow-lg` |
| 画面より大きい時 | 縦スクロール可（「次へ」が隠れない最終防御線） |

## 5. 文面のトンマナ（親しみやすく簡潔）

- **ですます調**・前向き・簡潔。専門用語を避け、アプリ内の**正式名称と一致**させる（例「AIツールを開く」「OKR管理」）。
- タイトルの**絵文字は1つだけ**・意味のあるもの。セット内で統一感を持たせる。
- **番号（①②③）は使うなら通し番号で連続させる**。②抜けの①③のような付け方はしない。使わないなら付けない。
- 各ステップは「**ここは何？ → 何ができる？ → 次にどうする？**」が分かること。
- **最初のステップ**：歓迎＋所要時間＋終了方法（✕/スキップ/Esc）＋再生方法（ガイドから）。
- **最後のステップ**：要点の振り返り＋困ったときの導線（？ボタン・ガイド）。
- 進捗（n/N）と終了手段は常時表示。

---

## 6. 振る舞い

- 暗幕クリックでは終了しない。終了は ✕ / スキップ / **Esc**。
- ターゲット付きステップは `skipIfMissing: true` を基本（UI変更耐性）。要素が無ければ次へ自動進行。
- リサイズ・スクロール時、および表示直後に再測位。
- `action` 付きステップ（実演）は、表示時に `window` の `tour:action` で通知し、アプリ側が副作用（パネルを開く等）を担う。実演でアプリ画面を操作・閲覧させる場合もスポットライトで対象を明示する（暗幕オフにはしない）。

---

## 7. アクセシビリティ（キーボード・スクリーンリーダー）

- 吹き出しは `role="dialog"` + `aria-modal="true"`、`aria-labelledby` でタイトルを関連付ける。
- ツアー開始時・ステップ変更時に**フォーカスを吹き出し（またはタイトル）へ移す**。Tab はツアー内のボタン
  （戻る／スキップ／次へ／✕）を巡回し、背後のアプリにフォーカスが抜けない（フォーカストラップ）。
- ステップ変更は `aria-live="polite"` 領域で読み上げ（「ステップ n / N：タイトル」）。
- キー操作：**Esc=終了 / →（または Enter）=次へ / ←=戻る**。入力欄にフォーカスがある実演ステップでは
  矢印キーを奪わない。
- ✕・各ボタンに `aria-label`。文字色・背景はコントラスト AA を満たす（ブランド色＋白文字は既存の組合せを使う）。
- **ダーク／ライト両テーマで成立**すること（色は必ずトークン経由。scrim の黒50%は両テーマで可）。

---

## 8. レスポンシブ・Teams 埋め込み

- 想定環境：PCブラウザ／Teams 埋め込み／（補助的に）モバイル。
- 画面幅が狭い時、吹き出しは `calc(100vw - 24px)` に収め、ターゲットに重ならない位置へ自動回避
  （`placement: "auto"` を基本に、収まらなければ中央）。
- PC専用UI（サイドバー等）を指すステップは `skipIfMissing: true`。対象が無ければ自動で次へ。
- Teams 埋め込みはウィンドウサイズが変わるため、リサイズで必ず再測位（実装済み）。位置は常にビューポート基準で算出。
- PC前提が多くモバイルで成立しにくいツアーは、無理に出さず最初のステップで「PCでの利用を推奨」と添えてよい。

---

## 9. ツアーの長さ・完了フラグ・再生

- **長さの目安：1ツアーあたり 7〜9 ステップを上限**（疲労防止）。超える場合はテーマで分割するか、
  後半を任意の続編ツアーにする。
- 1ステップの本文は §4（3行 or 箇条書き4点まで）を厳守。長くなるなら分割する。
- **完了フラグ**：`localStorage`（キー `tour_completed_v1`）で完了管理。**完了済みなら自動起動しない**。
- **自動起動**：初回ログイン時のみ「見ますか？」の招待を出す。スキップ／完了の選択は永続化し、
  再ログインで再表示しない。
- **再生・リセット**：ガイドのトップから手動再生できる（完了フラグの有無に関わらず）。再生は最初のステップから。

## 10. デザイントークン（単一の真実）

実装時に下記を一箇所に定義し、ハードコードを排除する（CSS変数 or 定数モジュール）。

```
--tour-scrim:      rgba(0,0,0,0.5);
--tour-ease:       cubic-bezier(0.16, 1, 0.3, 1);
--tour-duration:   0.2s;
--tour-spot-pad:   8px;          /* スポットライト余白 */
--tour-spot-radius: var(--radius-md);
--tour-spot-border: 2px solid var(--color-brand);
/* z-index: scrim 9990 / spotlight 9991 / bubble 10000 */
```

## 11. 新しいステップを作るときのチェックリスト

- [ ] `target`（data-tour-id）と `placement` を指定／中央なら省略
- [ ] `skipIfMissing: true`（ターゲット付きの場合）
- [ ] タイトルは絵文字1つ＋20字以内
- [ ] 本文は3行 or 箇条書き4点以内、「何→できる→次」の順
- [ ] アプリの正式名称と用語一致
- [ ] 番号を使うなら通し番号で連続
- [ ] 暗さ・余白・角丸・アニメは§1〜§3のトークンに従う（個別指定しない）

## 12. 実装フェーズで変更する対象（このガイドライン適用時）

- `src/styles/globals.css`：§10トークン＋吹き出し出現キーフレーム＋`prefers-reduced-motion` 対応を追加
- `src/components/tour/TourProvider.tsx`：
  - 二重暗幕の廃止／scrim トークン化（§1）
  - 吹き出しの出現・移動アニメ／くり抜きの明示遷移（§3）
  - `role="dialog"`・フォーカス移動・フォーカストラップ・`aria-live`・矢印/Enter/Esc キー操作（§7）
  - 狭い画面での吹き出し回避（§8）／スポットライト余白・角丸の統一（§2）／ヘッダー表記の統一（§4）
- `src/components/tour/tours/first-time.ts`：§4〜§5・§9に沿ってタイトル絵文字・番号・本文量・ステップ数を整える
- 完了フラグ・自動起動・再生（§9）は既存の `TourProvider` / `MainLayout` の挙動を基準に揃える
- ロジック・データ構造の本質的変更はなし（見た目・操作性・文面の統一が主）
