01

アプリ概要

グループ計画管理アプリは、チームのプロジェクト・タスク・OKRを リアルタイムで全員が共有・管理できるWebアプリです。 誰かが更新した内容は即座に全員の画面に反映され、 AIに変更の影響を相談したり、進捗を分析させることもできます。
社内でゼロから内製開発したアプリで、初期費用ゼロ・保守もAI支援で行っています。

PCブラウザから使える
複数人が同時編集しても競合しない
AIが変更の影響を整理・相談に応じる
Teamsと連携して通知を受け取れる
追加投資なし（無料枠で運用中）
02

技術スタック

Frontend
React + TypeScript
ブラウザ上で動く画面を構成する部品。操作への反応が速く、複雑な画面もコンポーネント単位で管理できる。
Database & Realtime
Supabase (PostgreSQL)
データを保存するクラウドDB。WebSocket通信で「誰かが更新した瞬間」を全員の画面にリアルタイムで届ける。
AI
Anthropic Claude API
タスク変更の影響整理・シミュレーション・OKR分析・レポート生成などAI機能全般を担う。
AI Security Layer
Supabase Edge Function
AIのAPIキーをサーバー側だけで保持するための中継役。ブラウザにキーが漏れない安全な構造を実現する。
Notification
Microsoft Teams Webhook
期限アラートやメンション通知をTeamsチャンネルに自動投稿。毎朝8時に定期チェックを実行。
Hosting & CI/CD
Vercel + GitHub
GitHubにコードを push した瞬間に本番環境へ自動デプロイ。インフラの管理作業は一切不要。

全体アーキテクチャ

04

データフロー — 3つのシナリオ

シナリオ ① タスクを更新したとき — リアルタイム同期
1
ユーザー A が画面でタスクを更新
ステータス変更・担当者変更・コメント入力など
2
Supabase の PostgreSQL に保存
更新前の日時（updated_at）と照合し、競合があれば警告を表示する「楽観的ロック」方式で多人数の同時編集を安全に処理
3
Realtime が変化を WebSocket で全員に配信
DB の変更が検知された瞬間、接続中の全ブラウザへ差分データを送信
4
ユーザー B・C の画面が即時に更新される
ページの再読み込みなしで変更が反映。全員が常に最新の状態を見ている
シナリオ ② AI に相談したとき — セキュアな中継
1
画面の「AI に相談」からメッセージを送信
タスク変更の影響範囲を整理、シミュレーション、進捗診断など 5 種類のモードから選択可
2
Supabase Edge Function（サーバー側）が受け取る
ブラウザではなくサーバーが次のステップを実行するため、APIキーが外部に漏れない
🔑 APIキーはサーバー内部にのみ存在
3
Edge Function が Anthropic の Claude API を呼ぶ
プロジェクト情報・タスク情報・相談内容をまとめて送信。個人名などセンシティブな項目はマスク処理済み
4
AI の回答が画面に表示される
「反映する」ボタンを押すと AI が提案した日程・担当者変更が実際のデータに適用される。会話履歴はセッション中のみ保持（DBに保存されない）
シナリオ ③ 期限アラート — 毎朝の自動通知
1
毎朝 8:00、Supabase の pg_cron が起動
DB に内蔵されたタイマー（pg_cron）が自動実行。外部サービスや手作業は不要
2
期限 3 日以内・超過のタスクを抽出
メンバーごとに「通知方法の設定」を参照し、「Teams」設定のメンバーの分をまとめる
3
Edge Function が Teams Webhook を呼び出す
対象タスク一覧を整形して Teams の指定チャンネルへ自動投稿
4
Teams チャンネルにアラートが届く
ブラウザ通知（リアルタイム）またはこの Teams まとめ通知を、メンバーごとに選択可能
05

主な機能

📋
プロジェクト・タスク管理
ガント・カンバン・リスト・カレンダーの4ビューを切り替えて表示。タスクに担当者・期日・優先度・親子関係を設定できる。
⚡
リアルタイム同期
誰かが更新した内容が即座に全員の画面に反映される。複数人が同時に編集しても競合を検知して安全に処理する。
🤖
AI に変更を相談
「影響範囲の整理」「What-If シミュレーション」「現状診断」「締め切り逆算」「スコープ縮小」の5モードでAIと対話できる。
🎯
OKR 管理
目標（O）・KR・タスクフォース・セッション記録・AI 分析・レポート生成まで OKR サイクルを一元管理。
🔔
期限アラート
メンバーごとに「ブラウザ通知」か「Teams まとめ通知」かを選択。毎朝 8 時に期限 3 日以内・超過タスクを自動通知。
💬
@メンション通知
コメント欄で @ を入力するとメンバー候補が表示され、選択するとメンションされたメンバーに通知が届く。

運用コスト感（概算・参考値）

項目 費用感 補足
Supabase ¥0 無料枠（Freeプラン）で運用中。DBストレージ 500MB・帯域 5GB/月まで無料。10人前後の社内利用には十分。
Vercel ¥0 個人・小規模チームは無料枠で運用可。商用・大規模利用時は Pro プランへ移行（$20/月〜）。
Claude API 従量課金 AI を使った分だけ課金。月の利用頻度によるが、社内 10 名規模では数百〜数千円/月程度が目安。
開発コスト 大幅削減 Claude Code（AI 支援開発ツール）を使ってゼロから内製開発。専門エンジニアへの外注費用が不要。
GitHub ¥0 個人・小規模チームは無料。プライベートリポジトリも無料枠内で利用可能。
07

同様のアプリを作るには

STEP 01
アカウント登録（3つ）
GitHub・Supabase・Vercel の 3 サービスに Web から登録するだけ。すべて無料で始められる。インストール作業は不要。
STEP 02
AI 支援開発で設計・実装
Claude Code などの AI 開発ツールを使えば、設計から実装まで非エンジニアでも対話しながら進められる。初期構築は数週間〜数ヶ月。
STEP 03
GitHub に push → 即デプロイ
コードを GitHub に push するだけで Vercel が自動でビルドし本番環境に反映。独自ドメインも簡単に設定できる。
⚠ 社内展開前に確認すべき事項
  • 保存するデータの機密レベルを確認し、Supabase への保管が社内情報セキュリティポリシーに適合するかを IT 部門に確認する
  • Claude API（Anthropic）にどんなデータを送信するかを整理し、社内の情報管理ルールとの整合性を確認する
  • API キーや認証情報の管理方法（Vercel / Supabase の環境変数）を理解し、GitHub に誤って公開しない運用ルールを定める
  • Anthropic の利用規約（コンシューマー規約 / 商用規約）を確認し、学習オプトアウトの設定と規約の適用範囲を把握する
