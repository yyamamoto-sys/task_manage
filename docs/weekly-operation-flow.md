# 週次運用フロー

> チェックイン・ウィンセッションを軸とした、1週間の標準的な運用の型です。  
> 「誰が・いつ・何をするか」の役割分担を明確にしておくことで、会議の質とアプリ活用が安定します。

---

## 1週間の全体像

```
月曜                火〜木               金曜
─────────────────────────────────────────────
チェックイン会議     タスク進捗の更新      ウィンセッション会議
（30分）            （各自・毎日数分）     （30〜45分）
  ↓                    ↓                    ↓
宣言・シグナルを      タスクのステータス    宣言達成確認・学び
アプリに記録          を随時更新            をアプリに記録
```

---

## 月曜：チェックイン

### 目的

今週の宣言（誰が何をいつまでにやるか）を全員で確認し、KRへの到達可能性を判断する。

### 役割分担

| 役割 | 担当 | やること |
|---|---|---|
| 書記（入力担当） | 1名（週番制推奨） | アプリへの入力・保存 |
| ファシリテーター | TFリーダーまたは固定担当 | 会議の進行 |
| 参加者全員 | 全員 | 宣言内容の確認・修正 |

### アプリでの手順

1. **ラボ → KRセッション記録** を開く
2. 対象KRを選択 → 「チェックイン」を選ぶ
3. 会議の文字起こし・議事メモを貼り付ける
4. 「AIで内容を抽出する」をクリック（30秒〜1分）
5. 抽出された内容を会議参加者と一緒に確認・修正
   - シグナル（🟢🟡🔴）：KR到達可能性の判断
   - 宣言リスト：誰が・何を・いつまでに
6. 「保存する」をクリック

> **ポイント**：抽出結果の確認は全員でやる。AIが誤読した宣言や抜け漏れをその場で直す。

### 所要時間の目安

| フェーズ | 時間 |
|---|---|
| 会議（宣言共有・シグナル確認） | 20〜25分 |
| アプリ入力・確認 | 5〜10分 |

---

## 火〜木：タスク進捗の更新

### 目的

タスクのステータスを常に最新に保ち、金曜のウィンセッションでズレなく振り返れるようにする。

### 役割分担

各メンバーが自分の担当タスクを随時更新する（1日1回程度が理想）。

### アプリでの手順

1. **ダッシュボード** または **カンバン** を開く
2. 担当タスクのステータスを実態に合わせて更新
   - 未着手 → 進行中 → 完了
3. 期限変更が生じた場合はタスク詳細から更新する

> **注意**：「完了のつもり」でなく実際に完了したものだけを「完了」にする。ウィンセッションでの宣言照合に使われます。

---

## 金曜：ウィンセッション

### 目的

今週の宣言達成状況を振り返り、学びを言語化して次週の打ち手につなげる。

### 役割分担

| 役割 | 担当 | やること |
|---|---|---|
| 書記（入力担当） | 1名（週番制推奨） | アプリへの入力・保存 |
| ファシリテーター | TFリーダーまたは固定担当 | 会議の進行 |
| 参加者全員 | 全員 | 達成状況の報告・学びの共有 |

### アプリでの手順

1. **ラボ → KRセッション記録** を開く
2. 対象KRを選択 → 「ウィンセッション」を選ぶ
3. 会議の文字起こし・議事メモを貼り付ける
4. 「AIで内容を抽出する」をクリック（30秒〜1分）
5. 抽出された内容を確認・修正
   - 宣言達成状況：達成 / 部分達成 / 未達成
   - 学び：仮説検証から何がわかったか
   - 外部環境変化：計画に影響する出来事
6. 「保存する」をクリック

> **ポイント**：未達成は責める場ではなく「なぜ達成できなかったか」を学ぶ場。原因を一言添えると次週の打ち手が具体化しやすい。

### 所要時間の目安

| フェーズ | 時間 |
|---|---|
| 宣言振り返り（達成・未達確認） | 15〜20分 |
| 学び・外部変化の共有 | 10〜15分 |
| アプリ入力・確認 | 5〜10分 |

## 月次（またはスプリント終了時）：KRレポート生成

チェックイン・ウィンセッションのデータが蓄積されたタイミングで、AIによるKRレポートを生成して共有します。

詳細は [KRレポート生成手順](./kr-report-howto.md) を参照してください。

## 運用上のポイント

### 書記の週番制を推奨する理由

特定メンバーに入力が集中すると、その人が不在のときに記録が止まります。週番制にすることでチーム全員がアプリ操作に慣れ、属人化を防げます。

### 文字起こしの取り方

AI抽出の精度は文字起こしの質に依存します。以下を参考にしてください。

| 方法 | 精度 | 手間 |
|---|---|---|
| Teams・Zoomの自動文字起こしを貼り付け | 高 | 低 |
| 議事メモ（箇条書き）を貼り付け | 中 | 中 |
| 会議後に記憶で手入力 | 低 | 高 |
